# 创建库

# 创建library

使用Angular Cli命令生产一个新库的骨架

# 语法
ng generate library 库名称 --prefix=前缀名
# 例如
ng generate library my-lib --prefix=zj
# prefix: 表示前缀

# 在library中创建一个新的组件

# 语法
ng generate component 组件名称 --project=库名称
# 例如
ng generate component button --project=my-lib

组件的代码

# 导出创建的组件

  • 在my-lib.module.ts中的exports对外暴露出ButtonComponent
@NgModule({
  declarations: [
    //...
    ButtonComponent
  ],
  imports: [
  ],
  exports: [
    //...
    ButtonComponent
  ]
})
  • 在public-api.ts中对外暴露出ButtonComponent
export * from './lib/button/button.component';

# 打包编译

  • 在项目的根目录找到projects节点,在该节点下找到创建的组件库my-lib,在architect.build下添加如下配置信息
{
  
  "configurations": {
    "production": {
       "project": "projects/库名称/ng-package.json"
     }
  }
}
  • 在项目的根目录下找到package.json文件,添加运行脚本
{

  "scripts": {
    "build_lib": "ng build --prod 库名称",
    "npm_pack": "cd dist/库名称 && npm pack",
    "package": "npm run build_lib && npm run npm_pack"
  }
}

使用npm run package运行脚本

在执行npm run package报了

ERROR: Unknown version 67 of android
Unknown version 67 of android
BrowserslistError: Unknown version 67 of android

直接将@angular/cli更新到最新,再重新上面的步骤

编译成功后会在dist/my-lib下生成my-lib-0.0.1.tgz文件

# 本地引用

将my-lib-0.0.1.tgz放入其他项目中,使用命令安装该库

# 语法
npm install 库名称路径

# 例子
npm install ./library/my-lib-0.0.1.tgz
